2: Enumerate

There are many times when we'll need to iterate over multiple lists in tandem. Here's an example:

xxxxxxxxxx
 
animals = ["Dog", "Tiger", "SuperLion", "Cow", "Panda"]
viciousness = [1, 5, 10, 10, 1]
for animal in animals:
    print("Animal")
    print(animal)
    print("Viciousness")

The above example, we have two lists, and the second list describes the viciousness of the animals in the first list. So a Dog has a viciousness of 1, and a SuperLion has a viciousness of 10. We want to get the position of the item in animals the loop is currently on, so we can use it to look up the corresponding value in viciousness.

Unfortunately, if we just loop through the animals list, there isn't a way to do this. Python has an enumerate() function that can help us with this, though:

xxxxxxxxxx
 
for i, animal in enumerate(animals):
    print("Animal Index")
    print(i)
    print("Animal")
    print(animal)

The enumerate() function allows us to have two variables in the body of a for loop -- an index, and the value. Here's a diagram:

On every iteration through the loop, i will be assigned the value of the index in animals that corresponds to the for loop iteration. animal will take on the value in animals that corresponds to the index i.

We can use the enumerate() function to iterate over multiple lists in tandem:

xxxxxxxxxx
 
animals = ["Dog", "Tiger", "SuperLion", "Cow", "Panda"]
viciousness = [1, 5, 10, 10, 1]
for i, animal in enumerate(animals):
    print("Animal")
    print(animal)
    print("Viciousness")
    print(viciousness[i])

In the above code, we use i to index the viciousness list, and print the viciousness value that corresponds to the same index in animals.

4: List comprehensions

We've written many short for loops to manipulate lists. Here's an example:

x
 
animals = ["Dog", "Tiger", "SuperLion", "Cow", "Panda"]
animal_lengths = []
for animal in animals:
    animal_lengths.append(len(animal))

It takes 3 lines to calculate the length of each string in animals. We can condense this down to 1 line with a list comprehension:

x
 
animal_lengths = [len(animal) for animal in animals]

The list comprehension above consists of the list operation len(animal), the loop variable animal, and the list that we're iterating over, animals.

A list comprehension condenses a for loop down like this:

Logically, the list comprehension is:

  • Looping through each element in animals and assigning the element to animal.
  • Finding the length of each animal string.
  • Generating a new list with all of the lengths as elements.
  • Assigning the new list to animal_lengths.

List comprehensions are much more compact notation, and can save space when you need to write multiple for loops.

6: None

Let's say we're trying to find the maximum value in a list. We might write some code that looks like this:

x
 
values = [50, 80, 100]
max_value = 0
for i in values:
    if i > max_value:
        max_value = i

We set max_value to a low value so everything is greater than it. But what if we changed the values list slightly?

xxxxxxxxxx
 
values = [-50, -80, -100]
max_value = 0
for i in values:
    if i > max_value:
        max_value = i

In the above scenario, max_value would be set to 0 when the loop finishes, which is wrong, because 0 isn't in values, it's just a placeholder we used to initialize max_value.

We can resolve this kind of issue using the None object. The None object has a special type called NoneType that differentiates it from other types.

The None object indicates that the variable has no value. Instead of using the normal double equals sign (==) to check if a value equals None, we use the variable is None syntax.

The is comparison operator checks for object equality. Using is instead of == prevents some custom classes from resolving to True when compared with None. We'll dive more into this in a later mission.

xxxxxxxxxx
 
values = [-50, -80, -100]
max_value = None
for i in values:
    if max_value is None or i > max_value:
        max_value = i

In the above code, we:

  • Initialize max_value to None.
  • Loop through each item in values.
  • Check to see if max_value equals None using the max_value is None syntax.
  • If max_value equals None, or if i > max_value, then we assign the value of i to max_value.
  • At the end of the loop, max_value will equal -50, which is in fact the largest value in values.

7: Comparing with None

Comparing a value to None will usually cause an error. This is helpful when we're writing code because it prevents unexpected variables from being None. This code will cause an error:

x
 
a = None
a > 10

When the Python interpreter evaluates a Boolean statement that contains or, like max_value is None or i > max_value, it will evaluate the statements in order. If the first statement is True, it won't evaluate the second statement. This is to save time, since if one statement is True, the whole or conditional is True.

Thus, when a value could potentially be None, and you want to compare it to another value, you should always check to see if it is None first. The below code will assign True to b if a is None or a is greater than 10:

x
 
a = None
b = a is None or a > 10

The same logic applies to an and statement. Since both and clauses have to be True, if the first one is False, the Python interpreter will not evaluate the second condition. The below code will not cause an error. It will assign True to b if a does not equal None and a is greater than 0:

x
 
a = None
b = a is not None and a > 10

9: The items method

In the last screen, it was slightly awkward to access both keys and values on dictionaries. We can simplify this process using the items() method, which allows us to iterate through keys and values at the same time.

xxxxxxxxxx
 
fruits = {
    "apple": 2,
    "orange": 5,
    "melon": 10
}
for fruit, rating in fruits.items():
    print(rating)

Using the items() method enables us to make our code more clear and compact.
